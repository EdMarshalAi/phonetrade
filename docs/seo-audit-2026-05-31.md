# Полный SEO-аудит PhoneTrade — 31.05.2026

Магазин техники Apple, Белгород (ул. Попова, 36). Домен `https://phonetrade31.ru`. Стек: Next.js 16 (App Router, standalone), Supabase. Цель: топ-1 по Белгороду (Яндекс-доминантный рынок) + AI/answer-engines.

**Метод:** 8 субагентов, по одной дименсии каждый, покрывают ~38 SEO-скилов по смыслу: технический SEO, мета/заголовки, Schema.org, контент/E-E-A-T, ключи/каннибализация, перелинковка/архитектура, локальный+AEO, изображения. Источники: исходный код репозитория + БД Supabase + проверка живого сайта.

> ⚠️ **Важно по методике:** на момент аудита сайт был в режиме «технических работ» — публичные страницы отдавали заглушку (HTTP 200, без `noindex`). Это (а) исказило часть live-проверок, (б) само по себе **критическая SEO-проблема** (см. ниже). Часть выводов сделана по коду — это источник истины.

## 1. Технический SEO

- **[Critical] Старый сайт по IP `http://31.129.97.8/` отдаёт 200 OK с полным контентом** — дубль. Проверено: `curl http://31.129.97.8/` → 200, тот же title/контент, без редиректа. Внутренние страницы по IP несут canonical на домен, но главная — нет. Рекомендация: на nginx для `Host = 31.129.97.8` отдавать `301` на домен либо `X-Robots-Tag: noindex`. Конфиг `/etc/nginx/sites-available/phonetrade`.
- **[High] У главной нет canonical-тега** (и `og:url`). Canonical задаётся только в `category/[slug]`, `product/[id]`, `blog/[slug]`, `trade-in`. Добавить `alternates:{canonical:"/"}` в `src/app/(site)/page.tsx`.
- **[High] Нет canonical на `/catalog`, `/new`, `/used`, `/blog` и катч-олле статических страниц** (`src/app/(site)/[slug]/page.tsx:18-22` — только title/description). Добавить self-referential canonical.
- **[Medium] Нет кастомной 404** (`not-found.tsx` отсутствует). Сам 404-статус корректен (честный 404), но экран дефолтный без шапки/поиска. Добавить `src/app/(site)/not-found.tsx`.
- **[Medium] Нет HSTS** (`Strict-Transport-Security`). Добавить на nginx после подтверждения стабильности HTTPS.
- **[Low] Директива `Host` в robots устарела** (`robots.ts:13`) — Яндекс упразднил, Google не поддерживает. Можно убрать.
- **[Low] Пагинация каталога в локальном стейте, не в URL** — для SEO скорее плюс (нет дублей `?page=`), но товары со 2-й+ страницы доступны боту только через sitemap. Sitemap покрывает все товары (608 URL), так что обнаруживаемость есть.
- **[Pass] robots.txt** корректен (Disallow `/admin /account /cart /auth /search`, sitemap указан, `/search` ещё и `noindex`). **[Pass] sitemap.xml** — 608 URL на домене, lastmod у всех, ISR 3600. **[Pass]** http→https и www→apex 301 работают, TLS валиден. **[Pass]** canonical товаров/категорий self-referential, фильтр-URL канонизируются на чистый путь. **[Pass]** `lang="ru"`, viewport, `next/font` swap, `next/image` remotePatterns.
- Замечание: статические маршруты в sitemap проставляют `lastModified: now` на каждый билд — «фейковая свежесть», лучше фиксировать дату изменения контента.

**Топ-3:** (1) закрыть/редиректить IP-дубль; (2) canonical на главную/коллекции/статику; (3) кастомная 404 + HSTS.

## 2. Мета-теги и заголовки

- **[Critical] Режим «тех. работ» → единственный H1 на всех URL «Идут технические работы»** (`src/app/(site)/layout.tsx:45-58`), заглушка с 200 без `noindex` → риск массовой деиндексации. Снять режим или отдавать 503+`Retry-After`.
- **[Critical] Двойной брендовый суффикс в title статики** — template `"%s · PhoneTrade"` (`layout.tsx:17`) + `meta_title` в БД уже с брендом → `/about`: «О компании — техника Apple в Белгороде | PhoneTrade · PhoneTrade» (бренд дважды, два разделителя). Использовать `title:{absolute}` для готовых title или хранить meta_title без бренда.
- **[High] Админ-модуль SEO (`seo_settings`) ни на что не влияет** — `SeoForm` пишет title_template/default_description/default_og_image/robots, но НИ один потребитель в `src` (вне модуля); значения захардкожены в `layout.tsx`/`robots.ts`. Подключить или удалить.
- **[High] Нет дефолтного `og:image`** нигде, кроме карточек товара → шаринг главной/категорий/блога без превью. Задать `openGraph.images` в `layout.tsx`.
- **[High] `twitter:card=summary`** (мелкое превью) глобально вместо `summary_large_image`. Задать в `layout.tsx`.
- **[High] Нет canonical** на главной/`/new`/`/used`/`/blog`/каталоге/статике (дубль с этим из §1).
- **[Medium] `/new` и `/used`: рассинхрон H1↔title, слабые title без города.** «Новинки Apple» (title) ↔ «Новинки» (H1). Обогатить: «Новинки Apple в Белгороде — iPhone, Mac, iPad».

- **[Medium] `og:title` категории = голое «iPhone»** без бренда/города (`category/[slug]/page.tsx:28`).
- **[Low]** title главной 48 симв. (ниже 50–60); поиск наследует OG главной (но noindex); H1 единственный на всех шаблонах (вне maintenance) — структурно ок.

**Топ-3:** (1) снять maintenance; (2) починить двойной суффикс + подключить/удалить seo_settings; (3) дефолтные og:image + twitter large_image + canonical.

## 8. Изображения (Image SEO)

- **[Critical] `unoptimized` на всей витрине** (Hero `:203`, Bento `:86,124`, BlogTeaser `:228,274`, TradeInPromo `:59`, ProductCard `:120`, ProductGallery `:46,72`) — нет WebP/AVIF/ресайза/srcset, исходники целиком на всех экранах. Логотип в Header (без unoptimized) проходит через `/_next/image` — механизм рабочий, отключён только для Storage-контента. Убрать `unoptimized`, настроить `next.config.images` (avif/webp, deviceSizes). Крупнейший выигрыш по LCP/CWV.
- **[High] Нет дефолтного OG/Twitter share-изображения** (`layout.tsx:28-34` без `images`) — шаринг любой страницы кроме товара без превью. Добавить 1200×630 + OG-обложки категорий/блога.
- **[High] Alt-тексты «общие»** — Hero `alt=slide.title`, Bento `alt=title` («iPhone»), TradeIn, ProductGallery главное фото `alt=product.title`. Обогатить гео: `Apple ${title} в Белгороде`, `${product.title} — купить в Белгороде`.
- **[Medium] ProductCard alt неплох** (`${title} — фото ${i+1}`) — для первого фото убрать «фото 1». **[Medium] LCP-картинки `priority` но `unoptimized`** — после снятия unoptimized заработает. **[Medium] Нет image-sitemap** (Next 16 поддерживает `images` в записи) — добавить обложки товаров/категорий/постов. **[Medium]** в `remotePatterns` остаются `picsum.photos`/`img.mvideo.ru` — убрать после миграции картинок (правило: только Storage).
- **[Low]** имена файлов не описательные (alt важнее); декоративные raw `<img>` с `alt=""` корректны; CLS под контролем (`fill`+`aspect-ratio`).

**Топ-3:** (1) снять `unoptimized` + avif/webp; (2) дефолтный OG/Twitter image + OG категорий/блога; (3) гео-alt + `images` в sitemap.
