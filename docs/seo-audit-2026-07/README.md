# SEO-аудит PhoneTrade — сводка (claude-seo, 02.07.2026)

Полный аудит `phonetrade31.ru` по набору **claude-seo** (25 скилов). Каждый скил — отдельный файл в этой папке. Метод: чтение методологии скила + проверка реального кода и живого сайта (curl/WebFetch). Цель — **топ по Белгороду в Яндексе (приоритет) + Google**.

**Общая оценка: техбаза сильная (~82/100).** HTTPS+HSTS, чистый SSR, богатая schema с `@id` (Product/Offer/Store/FAQPage/Breadcrumb/ItemList/WebSite), robots под Яндекс (Clean-param+Host), llms.txt, LocalBusiness с полным NAP и `areaServed` по области, гео-страницы городов. Проблемы — не в фундаменте, а в точечных дырах и в последствиях массового залива 12.06.

## 🔴 Критично (делать в первую очередь)

### Стабилизация после залива 12.06 (пока НЕ трогать массово)
- **Заморозить любые site-wide заливы** до отыгрыша Яндекса — каждый массовый ре-деплой сбрасывает таймер переоценки. Причина просадки = одномоментный дрейф `title/meta/description_html` у 317 товаров + категорий; baseline не снимался. → `seo-drift.md`, `seo-plan.md`.
- **Диагностика в Яндекс.Вебмастере**: переспам/Баден-Баден после залива, выпавшие страницы, 4xx/5xx за 12–13.06.
- **Завести `seo-drift baseline`** эталонных URL сейчас + сделать `drift compare` пред-деплойным гейтом (страховка от будущих регрессий). → `seo-drift.md`.

### Каннибализация блога (вероятная причина волатильности)
- **≥5 пар постов бьют в один запрос** (два «какой iPhone выбрать», два про MacBook, два про iPad, «Б/У или новый» ×2, несколько транзакционных «купить iPhone 17 Белгород»). Яндекс «мигает» между дублями → слить/301 в канонические URL. → `seo-content.md`, `seo-cluster.md`.

### Технические потери (быстрые, низкий риск)
- **IndexNow не внедрён** (`/indexnow` → 404). Поддерживают **Яндекс и Bing** → мгновенная переиндексация при смене цен/наличия. Точка врезки уже есть (`adminMutation` + `revalidatePath`). → `seo-technical.md`.
- **YML-фид отдаёт 9 товаров вместо ~300** (`/api/feed/yml`). `yml_feed_prefs.categories` режет выборку до аксессуаров/Dyson — iPhone/iPad/Mac/Watch/Samsung **не идут** в Яндекс.Товары и рекламу ВК. Чинится в админке за минуты. → `seo-ecommerce.md`.
- **LCP-картинка товара без `fetchpriority="high"`** — `priority` в коде есть, но в живом HTML теряется (вероятно `motion`/`AnimatePresence` в Next 16). Удар по LCP на 300+ карточках. → `seo-images.md`.
- **`opengraph-image.tsx` → 404** (динамический OG-баннер удалён, доки устарели). OG товара = голое фото без цены/бренда → слабый шеринг. Восстановить `next/og` (шрифт `_og/PTSans.ttf` на месте). → `seo-schema.md`, `seo-image-gen.md`.

### Локальный траст
- **Отзывов на сайте нет вообще** (0 `Review` в `src/`) — только badge-виджет Яндекс.Бизнеса. Trust — топ-фактор локалки. `aggregateRating` не делаем (решение владельца), но **контент отзывов** вывести стоит. → `seo-local.md`.
- **2ГИС и Apple Maps не в коде** — статически только Яндекс. 2ГИС для Белгорода — вторая карта, Apple Maps — под iPhone-аудиторию. `sameAs`-архитектура (`layout.tsx:54-58`) их уже поддержит. → `seo-maps.md`.

## 🟡 Важно (после стабилизации)

- **H1 категорий не локализован** (`/category/iphone` → H1 «iPhone» вместо «Купить iPhone в Белгороде») + **ломаная капитализация title** («IPad», «128gb», «м3») — бьёт по точным вхождениям (Яндекс чувствителен). → `seo-page.md`, `seo-technical.md`.
- **Sitemap-гигиена**: `/trade-in` дублируется дважды; noindex-товары (`is_indexable=false`) попадают в карту; `lastmod` = `now` (ложная свежесть). → `seo-sitemap.md`.
- **Домашний `<title>` без интента** — добавить «купить/цена/гарантия». → `seo-plan.md`, `seo-google.md`.
- **Offer без `shippingDetails` и `hasMerchantReturnPolicy`** → товары не проходят в Google Merchant free listings. → `seo-google.md`, `seo-ecommerce.md`.
- **Рассинхрон цены**: `meta_description` (47 000 ₽) vs Offer JSON-LD (51 000) — статик-мета устаревает при смене цены. → `seo-ecommerce.md`.
- **robots не управляет AI-краулерами** (нет директив GPTBot/YandexBot и т.д.) + **FAQ только на статических страницах** (нет на `/`, `/repair`, категориях — где локальные PAA). → `seo-geo.md`, `seo-local.md`.
- **Слабые/пустые кластеры** (наушники/часы/ноутбуки — по 1 посту; AirPods+Watch слиты в 1 URL) — источник «невидимости» из Вебмастера. 6 предложенных хабов + 2 готовых брифа. → `seo-cluster.md`, `seo-content-brief.md`.
- **E-E-A-T и структурные элементы**: нет автора-человека, `dateModified`=`datePublished`, нет «Похожих статей» (посты-сироты), нет FAQPage на категориях, нет страниц-сравнений «X vs Y». → `seo-content.md`, `seo-competitor-pages.md`.
- **На `/repair` нет прайса ремонта** — бьёт по SXO-персоне «ремонт». → `seo-sxo.md`.
- Проверить **дубль FAQPage** на карточке (из меморандума 13.06 — возможно уже устранён). → `seo-google.md`.

## 🟢 Nice-to-have / стратегия

- **Программатик услуга×город** (`/remont-apple-{город}`) — гео-страницы городов уже здоровый слой (~50% уникальности), масштабировать **батчами по 5–10**, наращивая уникальный блок (иначе thin). → `seo-programmatic.md`.
- **Отзывы на Яндекс.Картах/2ГИС** — самый дешёвый рычаг для локала и AI-видимости. → `seo-local.md`, `seo-maps.md`.
- **Ссылки — citations-first**: Яндекс.Бизнес + 2ГИС + Zoon/Flamp с консистентным NAP, без exact-match анкоров. Просадка 12.06 — не ссылочная. → `seo-backlinks.md`.
- **hreflang — задокументировать «не используем намеренно»** (моно-ru/моногео, состояние идеальное). → `seo-hreflang.md`.
- **DataForSEO — НЕ подключать**: всё покрыто бесплатно (Вордстат/Вебмастер/Topvisor). → `seo-dataforseo.md`.
