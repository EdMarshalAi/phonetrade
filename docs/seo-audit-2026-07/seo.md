# SEO-аудит: seo (orchestrator) — PhoneTrade

> Дата: 2026-07-02. Скил `seo` v2.2.0 — корневой оркестратор набора claude-seo.
> Оркестрирует 24 саб-скила (21 core + FLOW + 2 extension-зеркала) и до 18 саб-агентов.

## Что делает скил (карта набора)

`seo` — не аудитор сам по себе, а **диспетчер**. По команде `/seo audit <url>` он: (1) определяет тип бизнеса, (2) параллельно запускает саб-агентов, (3) собирает результаты в единый отчёт с SEO Health Score (0–100), (4) синтезирует через 10-принципную рамку PERCEIVE → ANALYZE → VALIDATE → ACT, (5) выдаёт приоритизированный план (Critical/High/Medium/Low) с зависимостями.

**Как весь набор работает вместе — по группам:**

| Группа | Скилы | Для нас |
|---|---|---|
| **Ядро аудита** | `seo-audit` (полный), `seo-page` (одна страница), `seo-technical` (9 категорий), `seo-content` (E-E-A-T) | техника закрыта C1–C8; используем `seo-page` точечно на регрессиях |
| **Схема/структура** | `seo-schema`, `seo-sitemap`, `seo-structure-architect` | Product/Offer/Org/Breadcrumb/Article/FAQPage есть; добрать OfferShipping/Return |
| **Контент** | `seo-content-writer`, `seo-content-brief`, `seo-cluster`, `seo-keyword-strategist`, `seo-meta-optimizer` | блог + мета; уже активно юзаем при заливах |
| **E-commerce** | `seo-ecommerce`, `seo-programmatic`, `seo-competitor-pages` | 317 товаров, варианты, категории — прямо наш профиль |
| **Локал** | `seo-local`, `seo-maps` | NAP/JSON-LD/карты — сильная сторона; отзывы — слабая |
| **GEO/AI** | `seo-geo`, `geo-fundamentals` | llms.txt, answer-first, FAQPage — фундамент готов |
| **Данные** | `seo-google` (бесплатно), `seo-dataforseo` (платно), `seo-backlinks` | Google — да; DataForSEO — нет (см. отдельные доки) |
| **Процесс** | `seo-flow` (цикл), `seo-drift` (регрессии), `seo-plan` (стратегия), `seo-sxo` | FLOW как регулярный пайплайн; **drift критичен после нашего залива** |
| **Медиа** | `seo-images`, `seo-image-gen`, `seo-hreflang` | фото в Storage с alt; hreflang не нужен (один язык/регион) |

**Синтез (10 принципов):** полный аудит проходит все фазы, узкие команды — минимум THINK + ACCEPT. Каждая рекомендация несёт: первопринцип, зависимость, «как поймём, что не сработало», leading-индикатор для мониторинга без ре-аудита. Это ровно то, чего нам не хватает после просадки — не «залить ещё», а гипотеза + проверяемость.

## Как применить к нам

Наш бизнес-тип оркестратор определит как **e-commerce + local hybrid** (сигналы: `/product`, `/category`, `add to cart` + телефон, адрес ул. Попова 36, Яндекс.Карты). Значит в полном `/seo audit` он поднимет: seo-technical, seo-content, seo-schema, seo-sitemap, seo-performance, seo-visual, seo-geo (всегда) + **seo-local + seo-ecommerce** (условно, по нашим сигналам) + seo-google (креды GSC есть) + seo-sxo (всегда).

**НО:** полный `/seo audit` сейчас противопоказан как действие — он ничего не деплоит, но соблазняет «починить всё сразу», а нам нельзя массово трогать сайт до отыгрыша Яндекса. Поэтому оркестратор используем как **карту**, а запускаем узкие команды точечно.

## Рекомендованный порядок для PhoneTrade (через оркестратор)

**Сейчас (стабилизация) — узкие команды, не полный audit:**
1. `seo-drift baseline` → зафиксировать текущее состояние SEO-элементов ДО отыгрыша. Потом `drift compare` покажет, что менялось. Это наш «git для SEO» — критично после 7 деплоев за 2 суток.
2. `seo-page` на 2–3 карточках не-iPhone (iPad/Mac/Watch) → подтвердить регрессию меты + проверить дубль FAQPage.
3. `seo-google` (Rich Results Test + GSC quick-wins) → см. `seo-google.md`.

**Через 1–2 недели (после отыгрыша) — добор:**
4. `seo-ecommerce` → Product-схема, OfferShipping/Return, Google Shopping eligibility.
5. `seo-local` + `seo-maps` → аудит NAP/GBP/отзывов (наш недокач).
6. `seo-content-brief` → брифы под оставшиеся «невидимые» запросы (точечно).

**Регулярно:**
7. `seo-flow` цикл (см. `seo-flow.md`) — еженедельно лёгкий, ежемесячно полный.

## Рекомендации

### 🔴 Критично
- **Не запускать полный `/seo audit` как повод для массовых правок** до отыгрыша Яндекса. Оркестратор — карта, не кнопка «пофиксить всё».
- **Завести `seo-drift baseline` немедленно** — чтобы ловить непреднамеренные регрессии при каждом деплое (после инцидента 12.06 это обязательно).

### 🟡 Важно
- Использовать 10-принципную рамку синтеза для КАЖДОЙ правки: гипотеза + «как поймём, что сработало». Хватит заливать без проверяемости.
- Приоритизация по Critical/High/Medium/Low из оркестратора — но с поправкой «не массово».

### 🟢 Nice-to-have
- Полный `/seo audit` прогнать ПОСЛЕ возврата позиций — как контрольный чекап, уже без риска.
- `seo-sxo` — почему хорошо оптимизированные карточки не в топе (intent-mismatch анализ).

## Быстрые победы
1. `seo-drift baseline` на главную + 3 категории + 3 карточки — 10 минут, страховка от будущих регрессий.
2. Использовать оркестратор-карту (эта таблица) как чеклист «что уже закрыто / что осталось».
3. Одна узкая команда за раз — не полный веер агентов.

## Google vs Яндекс
- **Оркестратор и весь набор Google-центричны** (INP, CrUX, AI Overviews, Merchant, Rich Results — всё про Google). Яндекс-специфику (Вордстат, Вебмастер, Баден-Баден, поведенческие, Алиса) набор напрямую НЕ покрывает — её мы ведём вручную + через мастер-план.
- **Что из набора работает на Яндекс:** seo-schema (Яндекс тоже читает JSON-LD), seo-local (карты/NAP/отзывы — критично для Яндекса), seo-content/E-E-A-T (Яндекс любит экспертность), seo-drift (регрессии агностичны), seo-geo (Алиса тянет структурированные факты).
- **Что чисто Google:** seo-google, seo-ecommerce (Merchant/Shopping), значительная часть seo-dataforseo.
- **Рабочее правило:** оркестратор + Google-скилы = Google-фронт; Яндекс.Вебмастер + Вордстат + Topvisor + отзывы на картах = Яндекс-фронт (приоритетный). Оба фронта питаются от одной схемы/контента, но замеряются раздельно.
