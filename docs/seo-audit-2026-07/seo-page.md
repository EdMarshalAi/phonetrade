# SEO-аудит: seo-page — PhoneTrade

_Дата: 2026-07-02. Метод: skill `seo-page` (глубокий разбор одной страницы). Разобраны: карточка товара `/product/[id]`, категория `/category/[slug]`, статик-страница `/[slug]`, `/repair`. Код + живой сайт._

## Что проверяет скил
On-page (title 50-60, description 150-160, ровно один H1, иерархия H2-H6, URL, внутр./внеш. ссылки), качество контента (объём, читабельность, плотность ключей, E-E-A-T, свежесть), тех-мета (canonical, robots, OG, Twitter Card, hreflang), schema (валидность, пропущенные типы; НЕ рекомендовать HowTo/FAQ-for-rich, FAQPage оставить как AI-сигнал), картинки (alt, размер, формат, width/height, lazy).

## Что у нас уже хорошо
- **Карточка товара — образцовая on-page структура.** Проверено на `/product/apple-ipad-air-11-m3-...`:
  - `<title>`: «... — Белгород · PhoneTrade» (гео + бренд).
  - self-canonical (`product/[id]/page.tsx:33`).
  - Ровно один H1 = название товара.
  - JSON-LD: `Product` + `Offer` (с `priceValidUntil`, `availability`, `itemCondition` New/Used, `seller @id`), `BreadcrumbList`, `FAQPage` — все четыре присутствуют вживую.
  - `robots: noindex` для `isIndexable===false` (`page.tsx:34`) — управляемая индексация.
- **Категория** (`/category/iphone`): title «Купить iPhone в Белгороде — цены, рассрочка, Trade-in» (коммерческий интент + гео), self-canonical, один H1, `BreadcrumbList` + `ItemList` (до 50 товаров) JSON-LD, SEO-текст под листингом (`seo_text` из БД, sanitized).
- **Статик-страницы** (`/[slug]`): canonical, `BreadcrumbList` + FAQPage из контента (`[slug]/page.tsx:53`), один H1 (или `<!--hide-title-->`).
- **`/repair`**: сильная локальная страница — title с гео и перечнем услуг, `Service` schema с `provider LocalBusiness @id`, `areaServed` Белгород.
- **OG/Twitter на карточке и категории** заданы (`page.tsx:35-36`), `type`, `images` из фото товара.
- **Картинки — Next `<Image>`** с `sizes`, `fill`, формат avif/webp; на карточке галерея, у блога заданы размеры (aspect-ratio контейнер → нет CLS).

## Находки / проблемы (конкретно, файл/URL)

1. **H1 категорий не локализован.** `/category/iphone` H1 = просто «iPhone» (проверено вживую), при том что `<title>` = «Купить iPhone в Белгороде...». H1 — сильнейший on-page сигнал; «iPhone» без гео/интента слабее, чем «iPhone в Белгороде» или «Купить iPhone в Белгороде». H1 берётся из `meta.title || cat.title` (`category/[slug]/page.tsx:54`) — это чистое имя категории.

2. **Ломаная капитализация в H1/title товаров** (дубль находки из seo-technical, но здесь это именно on-page-дефект): «Apple **IPad** Air 11 **м3**», «iPhone 13 128**gb**». H1 и title — ключевые поля; «IPad» вместо «iPad» бьёт по точному вхождению.

3. **Meta description на статик-страницах может быть пустой.** `[slug]/page.tsx:24`: `description: page.meta_description || undefined` — если у страницы не заполнен `meta_description`, тег не выводится вовсе, поисковик генерит сниппет сам. Стоит фолбэк на первые ~155 символов контента.

4. **Blog listing `/blog` — H1 «Блог», title «Блог о технике Apple».** H1 слишком общий; для темы «блог о технике apple в белгороде» H1 «Блог» не несёт ключей. Низкий приоритет (листинг).

5. **Description-длина не контролируется на рантайме.** meta_title ≤60 / meta_description ≤160 были подрезаны скриптом ранее (CLAUDE.md), но новые товары/категории из админки не имеют валидации длины в форме — риск дрейфа за 60/160.

6. **Внутренняя перелинковка карточки.** На карточке есть related (`getRelatedProducts`, 8 шт) + варианты — хорошо. Но нет явных текстовых ссылок «Ремонт этой модели» / «Trade-in» / «Б/У аналог» в теле — упущенная тематическая перелинковка (усиливает и /repair, и /used, и саму карточку).

7. **E-E-A-T на блоге слабоват.** `author: {"@type":"Organization","name":"PhoneTrade"}` (`blog/[slug]/page.tsx:50`) — организация как автор ОК, но нет `Person`-автора/эксперта, нет `dateModified` отдельно от `datePublished` (сейчас они равны — `page.tsx:49`). Для E-E-A-T и «свежести» полезен реальный dateModified.

## Рекомендации

### 🔴 Критично
- (нет блокирующих индексацию — страницы технически здоровы)

### 🟡 Важно
- **Локализовать H1 категорий.** В `category/[slug]/page.tsx` H1 сделать `${title} в Белгороде` (или отдельное поле `h1` в `categories`, фолбэк на «{title} в Белгороде»). Прямо усиливает главный кластер запросов «купить iphone белгород».
- **Фикс капитализации title/H1 товаров** (iPad/iMac/M-чипы/GB) — см. seo-technical, тот же скрипт закрывает и on-page.
- **Фолбэк meta_description** на статик-страницах и блоге: если пусто — брать первые 155 символов текста (обрезать по слову).
- **dateModified ≠ datePublished** в Article schema: писать реальную дату правки поста (`updated_at`), чтобы Яндекс/Google видели свежесть.

### 🟢 Nice-to-have
- Блок тематических ссылок в карточке товара: «Ремонт {модель}», «Trade-in», «Б/У {модель}» — усиление внутренней перелинковки к деньгам-страницам.
- Валидация длины meta_title (≤60) / meta_description (≤160) прямо в админ-форме товара/категории (счётчик символов) — чтобы не дрейфовало.
- H1 блога: «Блог о технике Apple» вместо «Блог».
- `Person`-автор в блоге для E-E-A-T (если есть реальный автор).

## Google vs Яндекс (нюансы под наш кейс — топ Белгород)
- **H1 с гео** для Яндекса весит больше, чем для Google: Яндекс сильнее любит точные вхождения «{товар} в Белгороде» в заголовках. Локализация H1 категорий — приоритетный ход именно под Яндекс.
- **FAQPage**: rich-сниппет FAQ Google ретайрил (май 2026), но **Яндекс FAQ-разметку продолжает показывать** и использует как сигнал → оставляем (скил тоже советует сохранить как AI/Яндекс-сигнал; не удалять).
- **Точная капитализация** («iPad», «M3») критичнее для Яндекса.
- **Description**: Яндекс чаще берёт description в сниппет как есть (Google переписывает) → пустой description на статик-страницах бьёт по Яндексу сильнее.
- **Свежесть (dateModified)** — Яндекс любит «живой» контент; корректный dateModified в блоге и реальный lastmod (см. seo-sitemap) работают вместе.
