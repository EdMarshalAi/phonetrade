# SEO-аудит: seo-plan — PhoneTrade

> Дата: 2026-07-02. Скил `seo-plan` v2.2.0 (шаблоны индустрий + 4-фазный roadmap).
> Контекст проекта: интернет-магазин Apple, Белгород, `https://phonetrade31.ru`, Next.js 16 + Supabase.
> Ключевой момент: **Яндекс просел 12–13.06 после массового SEO-залива** (см. `MEMORY/yandex-rank-drop-2026-06-13`).
> Отсюда — сначала СТАБИЛИЗАЦИЯ, потом РОСТ.

## Что проверяет/делает скил

`seo-plan` — стратегическое планирование: Discovery → Competitive Analysis → Architecture → Content Strategy → Technical Foundation → 4-фазный Implementation Roadmap (Foundation → Expansion → Scale → Authority). Подтягивает шаблон индустрии из `assets/` (для нас — `ecommerce.md` + `local-service.md`), выдаёт набор артефактов (SEO-STRATEGY, COMPETITOR-ANALYSIS, CONTENT-CALENDAR, IMPLEMENTATION-ROADMAP, SITE-STRUCTURE) и таблицу KPI-таргетов. Опционально усиливается DataForSEO для реальных цифр по конкурентам/объёму.

Наш кейс — гибрид: **e-commerce (транзакционный интент, товарные карточки, схема Product/Offer) + local-service (гео-запросы, карты, NAP, отзывы)**. Оба шаблона применимы одновременно.

## Как применить к нам

Классический roadmap скила — для сайта «с нуля». У нас техника уже закрыта (аудит C1–C8), 317 товаров, мета/схема/sitemap готовы. Поэтому фазы переосмысляем: **не строим фундамент заново, а стабилизируем после просадки и добираем контент под гэпы**.

Диагноз просадки честный: это не пенальти и не техсбой, а переоценка сайта Яндексом после 7 деплоев за 2 суток + возможный апдейт выдачи. Такое отыгрывается за 2–4 недели, если **не дёргать сайт массово**. Значит первая фаза плана — «замереть и точечно чинить», а не «залить ещё».

## Пошаговый SEO-план для PhoneTrade

### ФАЗА 0 — Стабилизация (сейчас, 1–2 недели) 🔴
Цель: дать Яндексу переварить залив, убрать регрессии, ничего массово не переписывать.

1. **Заморозить массовые site-wide правки.** Никаких «прогнать скрипт по всем 317 товарам». Только точечные правки конкретных URL. Каждый массовый ре-деплой = сброс таймера переоценки.
2. **Яндекс.Вебмастер — диагностика:** Безопасность и нарушения (фильтр Баден-Баден за переспам — вероятный риск после залива), Страницы в поиске (что выпало), Статистика обхода 4xx/5xx за 12–13.06, был ли общий апдейт выдачи. Это Фаза 0 из rank-drop-меморандума — выполнить до любых правок.
3. **Точечно починить 2 известные регрессии:**
   - Мета-описания не-iPhone (iPad, Mac, Watch, Яндекс.Станция) потеряли название модели → near-duplicate. Починить ПОштучно, не батчем.
   - Проверить дубль FAQPage на карточке (в коде сейчас один вызов `faqPageLd` — убедиться, что второй блок не приходит из `ProductDetailShell`/blocks; если дубля нет — снять из чеклиста).
4. **Мониторинг:** снимать позиции в Topvisor раз в 2–3 дня (регион = Белгород), фиксировать отыгрыш. Не реагировать паникой на дневную волатильность.

### ФАЗА 1 — Добор гэпов (2–4 недели) 🟡
Цель: закрыть запросы, где товар есть, а видимости нет (не трогая уже ранжирующееся).

5. **19 «невидимых» запросов при наличии товара** (playstation 5, стайлер dyson, samsung galaxy, наушники apple/airpods, ноутбук apple, смарт часы apple, модельные iPhone Pro) — это дыры в мете/контенте, не в ассортименте. Часть уже закрыта статьями от 25.06 (`seo-gap-content-2026-06-25.ts`) — **дождаться эффекта (1–2 нед), отправить новые URL на переобход**, потом добирать оставшиеся ТОЧЕЧНО.
6. **H1 категорий** — сейчас конкуренты дают голые названия («iPhone»); у нас возможность дать «Купить {категория} в Белгороде». Внедрять по 2–3 категории за деплой, смотреть реакцию (это site-structure-правка, аккуратно).
7. **Насыщенная H2-структура категорий** (Доставка/самовывоз ул. Попова 36 · Гарантия · Trade-in/Б/У · FAQ) — чего у конкурентов нет. Из БД (`seo_text`), не хардкод.

### ФАЗА 2 — Рост контентом и off-site (1–3 месяца) 🟢
8. **Блог 1–2 статьи/мес** под инфо+гео («какой iPhone выбрать», «сколько стоит 17 Pro Max в Белгороде», «как проверить Б/У»). Answer-first + FAQ + TL;DR. Уже есть паттерн (`seo-tldr-blog.ts`).
9. **Off-site / отзывы:** Яндекс Бизнес, 2ГИС, Google Business — карточки заведены (проверить единый NAP), просить реальные отзывы (QR на чеке). Для локала это сильнее, чем любой on-page.
10. **Ниша Б/У** (`/used` + «iPhone Б/У купить в Белгороде») почти пустая у конкурентов — усилить контентом.

### ФАЗА 3 — Авторитет (3–12 мес)
11. Гиперлокал (районы/ориентиры — никто не занимает), города области (Старый Оскол, Губкин), расширение ассортимента-семантики (Dyson, приставки, Samsung — конкуренты ранжируются, вектор совпадает).

## Google vs Яндекс
- **Яндекс (приоритет):** просадка — яндексовская. План Фазы 0 заточен под яндексовскую логику переоценки (медленный отыгрыш, чувствительность к массовым правкам, Баден-Баден за переспам). Гео-ключи + карты + отзывы решают.
- **Google:** массовый залив Google переносит спокойнее, санкций за это почти нет. Google быстрее переиндексирует, любит Product/Offer с shipping/return (Merchant free-listings — см. `seo-google.md`). Для Google фаза стабилизации не критична — там можно параллельно наращивать. Но приоритет владельца — Яндекс, поэтому темп задаёт он.
