# Дизайн-скилы проекта PhoneTrade

Полная карта дизайн-скилов: что входит в **обойму проекта**, для чего каждый и когда применять.

> **Правило проекта:** при ЛЮБЫХ изменениях дизайна прогонять **все** дизайн-скилы обоймы по очереди (см. авто-память `design-skills-rule`). Этот файл — каноничный список «всех» (сами скилы лежат вне гита, см. ниже).

## Где лежат скилы

- **`PhoneTrade/.agents/skills/<name>`** — обойма проекта (источник). Для «родных» скилов это реальные папки, для подключённых глобальных — симлинк на `~/.claude/skills/<name>`.
- **`PhoneTrade/.claude/skills/<name>`** — то, что читает Claude Code в проекте; относительный симлинк `→ ../../.agents/skills/<name>`.
- **`~/.claude/skills/` + `~/.agents/skills/`** — глобальные скилы (доступны во всех проектах). Все «родные» 13 дублируются туда же, поэтому они одновременно и проектные, и глобальные.
- ⚠️ Папки `.agents/` и `.claude/skills/` в проекте **в `.gitignore`** — в репозиторий не попадают. Поэтому состав обоймы фиксируется здесь, в этом файле.

**Подключить ещё один глобальный скил в обойму** (паттерн как у остальных):
```bash
ln -sfn "/Users/admin/.claude/skills/<name>" ".agents/skills/<name>"
ln -sfn "../../.agents/skills/<name>"        ".claude/skills/<name>"
```

## Обойма проекта (20 скилов)

### 1. Эстетика и «вкус» — визуальное качество
| Скил | Для чего | Когда применять |
|---|---|---|
| **minimalist-ui** | Чистый редакторский UI: тёплая монохромная палитра, типографический контраст, плоские bento-сетки, без градиентов и тяжёлых теней | **Базовый для PhoneTrade** — наша Apple-эстетика (серо-чёрная палитра) |
| **high-end-visual-design** | «Дорогой» вид как у топ-агентства: точные шрифты, отступы, тени, структура карточек, анимации; блокирует дешёвые AI-дефолты | Любая новая страница/секция, которая должна выглядеть премиально |
| **design-taste-frontend** | Anti-slop фронтенд: читает бриф, выбирает направление, делает не-шаблонные интерфейсы; audit-first на редизайнах | Лендинги, новые разделы, редизайн |
| **design-taste-frontend-v1** | Старая v1 того же скила (для обратной совместимости) | Только если нужно точное поведение v1; по умолчанию — v2 выше |
| **gpt-taste** | UX/UI + продвинутый GSAP-моушн: рандомизация раскладки, AIDA-структура, широкая редакторская типографика, bento без зазоров, ScrollTrigger | Когда нужен сложный скролл-сторителлинг и моушн |
| **stitch-design-taste** | Семантическая дизайн-система (Google Stitch): генерит `DESIGN.md` с премиальными анти-генерик стандартами | Когда нужен формализованный design-spec для секции |
| **industrial-brutalist-ui** | Брутализм: швейцарская типографика + терминальная эстетика, жёсткие сетки, аналоговые дефекты | ⚠️ Противоположен нашему стилю — для контраста/экспериментов, в проде обычно не используем |
| **frontend-design** | «Дизайнер-инженер, а не генератор раскладок»: именованное эстетическое направление + production-код, DFII-оценка | Когда нужно одновременно задать стиль и сразу выдать рабочий код |

### 2. UI/UX-ревью и принципы
| Скил | Для чего | Когда применять |
|---|---|---|
| **ui-ux-pro-max** | Большой справочник: 50+ стилей, 97 палитр, 57 пар шрифтов, 99 UX-правил, 25 типов графиков | Выбор палитры/типографики, ревью кода на UX-проблемы |
| **uxui-principles** | Оценка интерфейса против 168 research-backed UX/UI-принципов, детект антипаттернов | Аудит интерфейса, инъекция UX-контекста в сессию |
| **ux-heuristics** | Юзабилити по эвристикам Нильсена, severity-рейтинги, информационная архитектура | «Юзеры путаются», аудит форм/навигации, usability-review |
| **web-design-guidelines** | Проверка файлов на соответствие Web Interface Guidelines | Финальная сверка вёрстки с веб-гайдлайнами |

### 3. Дизайн-система и стили
| Скил | Для чего | Когда применять |
|---|---|---|
| **tailwind-design-system** | Production design-system на Tailwind: токены, варианты компонентов, адаптив, доступность | **Релевантно — у нас Tailwind v4 + CVA**; новые компоненты/токены |
| **theme-factory** | Курированные темы: палитры + пары шрифтов, применяются к артефакту | Быстрый подбор/смена темы (цвет + шрифт) |

### 4. Бренд и генерация графики
| Скил | Для чего | Когда применять |
|---|---|---|
| **brandkit** | Премиальная генерация бренд-бордов, лого-систем, identity-деков | Бренд-материалы, гайдлайн-борды |
| **image-to-code** | Сначала сгенерировать дизайн-картинку, проанализировать, затем сверстать 1:1 | Визуально важные секции «от картинки к коду» |
| **imagegen-frontend-web** | Генерация референсов веб-секций: ОДНА горизонтальная картинка на КАЖДУЮ секцию | Подготовка дизайн-референсов лендинга перед версткой |
| **imagegen-frontend-mobile** | Генерация app-native экранов в премиальном телефонном мокапе (iOS/Android) | Концепты мобильных экранов/флоу (только картинки) |

### 5. Процесс и утилиты
| Скил | Для чего | Когда применять |
|---|---|---|
| **redesign-existing-projects** | Апгрейд существующих сайтов до премиума: аудит → детект generic AI-паттернов → high-end без поломки функционала | Редизайн готовых страниц проекта |
| **full-output-enforcement** | *(не дизайн)* запрет на обрезку вывода, полная генерация кода без плейсхолдеров | Большие правки, где важна неурезанная выдача |

## Самое релевантное для PhoneTrade

Стек — Next.js 16 + Tailwind v4 + @base-ui-components, строгая Apple-эстетика (серо-чёрная палитра). Приоритетные при дизайн-правках:

1. **minimalist-ui** — базовый стиль витрины.
2. **high-end-visual-design** — «дорогой» вид.
3. **tailwind-design-system** — токены/компоненты на нашем Tailwind v4.
4. **ui-ux-pro-max** + **uxui-principles** + **ux-heuristics** — выбор палитры/типографики и UX-ревью.
5. **web-design-guidelines** — финальная сверка.
6. **design-taste-frontend** / **frontend-design** — для новых разделов и лендингов.

Обычно **не** применяем: `industrial-brutalist-ui` (противоположный стиль), `design-taste-frontend-v1` (legacy).
