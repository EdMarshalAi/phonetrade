# План по SEO — мастер-чеклист для будущих проектов

> Полная карта действий по SEO/GEO, которую мы прошли на **PhoneTrade** (интернет-магазин
> техники Apple, Белгород, Next.js 16 + Supabase). Документ написан так, чтобы по нему
> можно было **с нуля настроить SEO на другом проекте** — по порядку, со ссылками на наши
> файлы-образцы, источники ключей и конкретные настройки.
>
> Читать сверху вниз. Каждый раздел — отдельный этап. В конце — быстрый чеклист и календарь.

**Легенда:** 🟢 делается один раз · 🔁 повторяется/поддерживается · 📁 наш файл-образец · 🔗 внешний сервис

---

## 0. Принципы (с них начинается всё)

1. **NAP-консистентность.** Название / адрес / телефон ОДИНАКОВЫ везде: сайт, schema, Яндекс.Карты,
   2ГИС, соцсети, каталоги. Любое расхождение → поисковик не «склеивает» сущность.
2. **Гео в каждом ключе.** Для локального бизнеса коммерческий запрос почти всегда = `{интент} + {товар} + {город}`.
   Цель — топ-1 по городу, а не по РФ (там не выиграть и трафик нецелевой).
3. **Один источник истины для контента.** Мета и SEO-тексты — в БД/админке, не хардкод. Тогда новая
   страница/товар автоматически попадает в sitemap и получает мету.
4. **Не выдумывать.** Цены, статистику, отзывы, гарантии — только реальные. Конкурентов в текстах не упоминать.
5. **Технику закрыть на 100%, дальше — off-site и время.** Локальное SEO/GEO — это недели, не дни.

## 1. Технический фундамент (🟢 один раз)

Без этого остальное не индексируется. Проверять в таком порядке:

| Что | Зачем | Где у нас |
|---|---|---|
| **HTTPS + один канонический хост** | дубли `www`/`http`/IP убивают вес | nginx: `www→apex`, `http→https` 301, IP→301 на домен |
| **`sitemap.xml`** динамический, ISR | поисковик видит все URL, обновляется сам | 📁 `src/app/sitemap.ts` (главная, категории, товары **с `<image>`**, static_pages, блог; `revalidate=3600`) |
| **`robots.txt`** | allow контент, disallow служебное | 📁 `src/app/robots.ts` (disallow `/admin /account /cart /auth /search`) |
| **Canonical + метатеги на каждой странице** | антидубли, сниппет | `generateMetadata()` в каждом `page.tsx` (Server Components) |
| **og:image баннер 1200×630** | красивый шер в соцсетях/мессенджерах | 📁 `src/app/opengraph-image.tsx` (`next/og`, статический шрифт — вариативные Satori не умеет) |
| **Скорость / Core Web Vitals** | фактор ранжирования | Next Image (avif/webp), `output: standalone`, ISR-кэш |
| **Мобильная вёрстка** | mobile-first индекс | адаптив на всех страницах |

**Правило для Next.js:** страницы — Server Components, данные через геттеры, `"use client"` только на интерактиве.
Метаданные генерятся на сервере из БД → SEO-поля редактируются в админке без релиза.

## 2. Микроразметка Schema.org / JSON-LD (🟢 один раз, 🔁 расширяется)

Это «язык», на котором сайт объясняет поисковику и нейросети, что он такое. Внедрили:

| Schema | Где | Что даёт |
|---|---|---|
| **Organization / Store (LocalBusiness)** | 📁 `src/app/(site)/layout.tsx` — `orgLd` | карточка организации, знание-граф, привязка к городу |
| ↳ `address`, `geo`, `openingHoursSpecification` | там же | адрес/координаты/часы в выдаче и на картах |
| ↳ `areaServed` (город + область + города области) | там же | «обслуживаем Старый Оскол, Губкин…» |
| ↳ `knowsAbout` (бренды/категории) | там же | тематическая привязка для ИИ |
| ↳ **`sameAs`** (соцсети + Яндекс.Карты + 2ГИС) | там же, фильтр по regex href | связывает сайт с картами/соцсетями = доверие |
| **Product / Offer** | страница товара | цена, наличие, `priceValidUntil` → rich snippet с ценой |
| **BreadcrumbList** | категории/товар | хлебные крошки в выдаче |
| **Article** (+ publisher → `#organization`) | страница блога | дата, автор, картинка в сниппете |
| **FAQPage** | товар, блог, static-страницы | блок «вопросы-ответы» прямо в выдаче + цитируемость в ИИ |

**Как мы делали FAQ-схему автоматически:** 📁 `src/lib/utils/faq-schema.ts` — `faqFromHtml()` вытаскивает
Q/A из секции после маркера «Частые вопросы» в HTML описания (с ограничениями длины), `faqPageLd()`
строит JSON-LD. Вшито в товар/блог/статичную страницу. → не надо руками размечать каждую страницу.

**`sameAs` — как добавлять карты:** контакты лежат в `shop_settings.general.contacts` (массив `{id,href,icon,label}`).
Layout фильтрует href по regex (`yandex.ru/maps`, `2gis.ru`, `vk.com`, `t.me`…) и кладёт в `sameAs`.
⚠️ Проверяй, что карточка карт — **твоего города** (у нас была подсунута карточка из Геленджика — отловили по
координатам/городу в URL; правильную белгородскую добавили `2gis.ru/belgorod/firm/...`).

## 3. Семантика: откуда брать ключи (🟢 сбор, 🔁 пополнение)

Это сердце SEO. Порядок сбора:

### 3.1 Источники ключей (что мы реально использовали)
1. **Яндекс.Вордстат** — основной для РФ. Выгрузка «Рекомендованные запросы» и «Tasks/KeywordsResearch»
   (CSV на 20k+ строк). Это реальная частотность по региону.
2. **Семантика конкурентов** — выгрузка их запросов из сервисов (у нас был CSV с позициями).
   Разведка постранично: их H1, заголовки, какие config-страницы у них в индексе.
3. **Подсказки/«вместе ищут»** в Яндексе и Google по основным запросам.
4. **Структура каталога** — каждая модель × память × цвет × «купить/цена/бу» = валидный ключ.

### 3.2 Обработка (как чистили)
- Кодировки CSV из выгрузок: UTF-16LE или CP1251 → `iconv` в UTF-8.
- Убрать мусор: чужие бренды-магазины, нерелевантные города, нулевую частотность, дубли.
- **Кластеризация** (скилл `seo-keyword-strategist`): группы «модель», «память», «Б/У», «trade-in»,
  «рассрочка», «ремонт», «города области», «бренды-смежки».
- Разговорные/транслит-формы: `айфон/iphone/афон`, `про макс/pro max/промакс`, память `256гб/256 гб/256`.

### 3.3 Что родилось (наши файлы-образцы)
- 📁 `docs/seo-keywords-all.csv` — весь массив (9000+ после чистки)
- 📁 `docs/seo-keywords-belgorod.csv` — гео-срез
- 📁 `docs/seo-keywords-plan.md` — план: какие кластеры куда вшиваем
- 📁 `docs/seo-competitors-recon-2026-06.md` — разведка конкурентов (H1, структура, выводы)
- 📁 `docs/topvisor-keywords.txt` — 125 ключей для **отслеживания позиций** (см. §8)

**Вывод по «весу запросов» конкурента:** большой «вес» в Яндексе ≠ трафик. У конкурента 1185 запросов,
но 743 на позициях 31–50 (ноль трафика), в топ-10 только ~48. Секрет «массы» — много **индексируемых
страниц-конфигураций** (модель+цвет+память) и разговорные варианты. Стратегия-ответ: больше таких
страниц у себя + разговорная семантика в текстах.

## 4. Вшивание ключей в сайт (🔁 при каждом новом товаре/категории)

Куда именно идут собранные ключи:

### 4.1 Товары (у нас 332 шт, скриптом)
📁 `scripts/seo-embed-keywords.ts` (идемпотентный, маркер `<!--seo-kw-->`):
- `meta_title` ≤ 60 символов: `{Модель} {память} {цвет} купить в Белгороде — цена | Бренд`
- `meta_description` ≤ 160: цена + память по-русски + интент + гео + CTA + «✓ гарантия».
- В `description_html` дописывает блок: разговорные формы модели, RU-память, мини-FAQ (для FAQPage-схемы).
- Уникальный per-unit лид-абзац, чтобы не было дублей контента между похожими товарами.

### 4.2 Категории (52 шт)
📁 `scripts/seo-embed-categories.ts`: дописывает в `seo_text` гео-блок (город + города области) и
ключи кластера категории. Мета категории — `meta_title`/`meta_description` из админки (`getCategoryMeta()`).

### 4.3 Правила
- Плотность ключа 0.5–1.5%, без переспама (скилл `seo-meta-optimizer`).
- Гео — естественно, не «купить айфон Белгород Белгород Белгород».
- **Категории/мета — строго из БД/админки.** Никаких хардкод-конфигов на витрине.

## 5. Контент-маркетинг: блог (🔁 регулярно)

Блог закрывает информационные и низкочастотные гео-запросы + кормит GEO (ответы ИИ).

- **Темы — из семантики:** «какой iPhone выбрать», «сколько стоит 17 Pro Max в Белгороде»,
  «как проверить Б/У iPhone», «trade-in в Белгороде», «где купить» и т.п.
- **Формат под SEO+GEO:** вопрос в заголовке, **TL;DR/«Кратко» первым абзацем** (answer-first для ИИ),
  H2/H3, таблицы, FAQ в конце (→ FAQPage-схема), E-E-A-T (проверка по серийнику, гарантия, сервис).
- Писали пачкой через сабагентов (скилл `seo-content-writer`), обложки — генерацией изображений.
- 📁 `scripts/seo-tldr-blog.ts` — добавляет блок «Кратко» в начало статей (idempotent, маркер `<!--tldr-->`).
- **Счётчики просмотров** (соц.доказательство): RPC `increment_blog_views`, показ на карточке и в статье.

**Объём у нас:** ~25 статей. Правило — лучше реже и качественнее, чем спам.

## 6. Локальное SEO и карточки-агрегаторы (🟢 завести, 🔁 отзывы)

Сайт сам по себе в локальном поиске не победит — нужны **внешние карточки** (off-site).
📁 готовый текст для всех карточек: `docs/business-listings-content.md` (NAP + описание + услуги — копировать без изменений).

Порядок заведения:
1. 🔗 **Яндекс Бизнес** (Карты/Нейро/Алиса) — `business.yandex.ru`. Главное для РФ. → даёт `yandex.ru/maps/org/...`
2. 🔗 **2ГИС** — `account.2gis.com`. → `2gis.ru/{город}/firm/...`
3. 🔗 **Google Business Profile** (если доступно) — кормит Google Maps и AI Overviews.
4. 🔗 **Яндекс Вебмастер** — верификация (метатег в админку/`<head>`) + отправить sitemap.
5. 🔗 **Google Search Console** — верификация + sitemap.
6. 🔗 **Bing Webmaster** — ПЕРВЫМ ДЕЛОМ для ChatGPT (без Bing в ChatGPT не попасть). Sitemap туда же.
7. **Соцсети** (VK, Telegram) — указать сайт, постить товары/статьи; профили → в `sameAs`.
8. **Ссылки карт → в `sameAs`** на сайте (см. §2) — двусторонняя связка сайт↔карта.

**Отзывы = доверие + цитируемость в ИИ.** Просить покупателей оставлять отзывы на Яндекс.Картах и 2ГИС
(QR/ссылка на чеке). 10–20 реальных отзывов заметно поднимают и локал, и видимость в нейросетях.

⚠️ **Платные ссылки/прогоны (Kwork и т.п.)** — не покупали и не советуем: для локального магазина риск
фильтра выше пользы. Лучше живые карточки + отзывы + контент.

## 7. GEO / AEO — попасть в ответы нейросетей (🟢 техника, 🔁 контент)

GEO = Generative Engine Optimization (цитирование в ChatGPT/Perplexity/Алиса/AI Overviews). Что внедрили:

- **`llms.txt`** в корне — карта сайта «для ИИ»: NAP, категории, гиды. 📁 `public/llms.txt`.
- **Доступ ИИ-краулерам** разрешён в robots (GPTBot, PerplexityBot, Claude-Web, Googlebot).
- **Answer-first контент:** TL;DR/«Кратко» вверху статей, чёткие определения, FAQ — то, что ИИ легко извлекает.
- **Schema FAQPage/Article/Organization** (§2) — структурированные факты для RAG.
- **Off-site присутствие** (§6) — нейросети тянут факты из карт/каталогов/отзывов.
- **Bing Webmaster** — критично для ChatGPT/SearchGPT.

Что НЕ даёт мгновенного эффекта и зависит от времени: реальные цитирования появляются через недели
после индексации карточек и накопления отзывов. Технически — мы готовы; дальше off-site + время.

Скиллы-помощники: `geo-fundamentals`, `seo-aeo-landing-page-writer`.

## 8. Отслеживание роста (🔁 постоянно)

Без замера позиций не понять, растёшь ли. Настройка:

- 🔗 **Topvisor** (или аналог) — съём позиций. 📁 список ключей: `docs/topvisor-keywords.txt` (125 шт).
  - Импорт: Ключевые слова → добавить списком → вставить файл.
  - **Регион поиска: твой город** (у нас Белгород) в Яндексе и Google — иначе гео-ключи мерятся неверно.
  - Поисковики: Яндекс + Google. Съём раз в 2–3 дня.
  - Смотреть динамику по группам: бренд (должен быть #1), гео-ключи (наш рынок), города области (быстрый топ).
- 🔗 **Яндекс.Метрика** — трафик/поведение (у нас с согласием на cookie, многоуровневый сбор).
- 🔗 **Вебмастеры** (Яндекс/Google/Bing) — индексация, ошибки, запросы-показы.
- **Внутренняя аналитика** — `page_views` с фильтром ботов (важно: боты раздувают цифры в разы; у нас
  было 100k+ «визитов», из них ~98% боты — обязательно фильтровать `is_bot`).

## 9. Юридический минимум (🟢 один раз) — чтобы аналитика была легальной

SEO тянет за собой сбор данных → нужно прикрыть по закону (РФ 152-ФЗ + закон о cookie):

- **Cookie-баннер** анти-дарк-паттерн: 3 равнозначные кнопки (Принять все / Только необходимые / Настроить),
  нейтральный текст, лёгкое затемнение экрана. Согласие в first-party cookie на 13 мес.
- **Многоуровневый сбор аналитики по согласию:** Level 0 (обезличенно всегда) → Level 1 (с согласием) →
  Level 2 (реклама). Метрика грузится после согласия (или по флагу «собирать без согласия» — на усмотрение).
- **Политика конфиденциальности + Согласие на обработку ПДн** — реальные документы (у нас в `static_pages`).
- Галочка согласия на ПДн в формах; принятие оферты — текстом под кнопкой, не отдельной галочкой.
