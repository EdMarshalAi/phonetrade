# Промт для Claude Code — Дашборды «Аналитика сайта» и «Аналитика заказов» для PhoneTrade

> Скопируй этот файл в проект (`docs/analytics-dashboards.md`) и используй как промт для Claude Code.
> Задача — за один заход переделать обе аналитики на уровень зрелого e-commerce: с табами, понятными подсказками для не-маркетолога, сравнением с прошлым периодом и качественной визуализацией.

---

## 0. ЦЕЛЬ И КОНТЕКСТ

У нас есть рабочий магазин Apple-техники PhoneTrade (Белгород) на Next.js 16 + Supabase. В админке уже есть **базовые** дашборды `/admin/analytics/site` и `/admin/analytics/orders` — но они слабые: статичные плитки без сравнения, пустые блоки, нет глубокой аналитики поведения и продаж, нет ABC/XYZ-анализа товаров, нет LTV/RFM по клиентам, нет когорт, нет тултипов-подсказок.

**Что нужно сделать:**
- Полностью пересобрать оба дашборда с разбивкой по тематическим **табам** внутри каждого раздела
- Каждая метрика снабжена **тултипом-подсказкой** с человеческим объяснением (магазин ведёт владелец без маркетингового бэкграунда — он должен понимать каждую цифру)
- Реализовать **сравнение с прошлым периодом** на всех KPI
- Премиальный, чистый, плотный, читаемый дизайн в стиле Apple HIG
- **Всё на русском языке** — заголовки, метрики, тултипы, empty states

- `tailwind-design-system`, `theme-factory` — токены и темы
- `shadcn`, `baseline-ui` — примитивы на base-nova / @base-ui/react

### Скилы по фазам (доп. к обязательным):

| Фаза | Доп. скилы | MCP |
|---|---|---|
| 1. Фундамент данных и компонентов | `core-components`, `interface-design` | `ide` (TS-диагностика) |
| 2. Аналитика сайта | `kpi-dashboard-design` (ключевой!), `scroll-experience` | `magic` для черновых вариантов компонентов |

## 2. ДИЗАЙН-ПРИНЦИПЫ ДАШБОРДОВ

### 2.1. Структура страницы (одинаковая для обеих аналитик)

```
┌─────────────────────────────────────────────────────────────┐
│ Шапка страницы                                              │
│ ┌──────────────────────────┬──────────────────────────────┐│
│ │ Аналитика сайта          │ [Период: 30 дней ▼] [Срав-нь]││
│ │ Краткое описание периода │ [Экспорт ▼]                  ││
│ └──────────────────────────┴──────────────────────────────┘│
├─────────────────────────────────────────────────────────────┤
│ [Обзор] [Аудитория] [Поведение] [Источники] [Технические] │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Контент выбранного таба                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

- **Sticky-шапка** с пикером периода — при скролле не уезжает
- **Табы** — горизонтальная навигация в стиле Apple (snake-подложка под активным элементом с плавной анимацией, через `motion`)
- **URL отражает таб:** `?tab=audience` — чтобы шарить ссылки на конкретный таб

### 2.2. Глобальный фильтр периода (`PeriodPicker`)

Sticky-блок в шапке страницы. Содержит:
- Кнопки-пресеты: **Сегодня / 7 дней / 30 дней / 90 дней / Квартал / Год / Произвольно**
- Произвольно → открывается date-range picker
- Toggle **«Сравнить с прошлым периодом»** (включён по умолчанию)
- При выборе показывает в подзаголовке: «Период: 1–30 мая 2026. Сравнение с 1–30 апреля 2026»
- Состояние пикера хранится в URL (`?from=...&to=...&compare=true`)

### 2.3. KPI-плитки (`KpiCard`)

Это главный визуальный элемент — на нём пользователь задерживается дольше всего.

**Структура плитки:**
```
┌─────────────────────────────┐
│ ВЫРУЧКА ⓘ            +12.4% │  ← подпись (uppercase, серый, мелкий) + дельта (зелёная/красная)
│                              │
│ 1 245 000 ₽                  │  ← главное число (крупно, font-semibold, tracking-tight)
│                              │
│ ▁▃▅▇▆▇█▇▆▅▆▇█                │  ← sparkline за период (тонкая линия)
│                              │
│ vs 1 108 000 ₽ за прошлый    │  ← мелкая подпись со сравнением
└─────────────────────────────┘
```

**Правила:**
- Иконка `Info` (lucide) рядом с подписью → hover/tap = тултип с пояснением (см. раздел 7 «Глоссарий»)
- Дельта в правом верхнем углу: зелёная если рост и это хорошо, красная если падение и это плохо. **ВНИМАНИЕ:** для метрик-«меньше=лучше» (bounce rate, отмены, drop-off) логика инвертируется → рост красный, падение зелёное
- Sparkline — без осей, без подписей; цвет — текущий accent (чёрный); фон плитки белый
- При отсутствии данных за прошлый период — показывать «—» вместо процента
- При нулевых данных — показывать «0» и подпись «нет данных за период», но плитку не скрывать

### 2.4. Тултипы-подсказки (`MetricTooltip`)

**Ключевая фишка для не-маркетолога.** Каждая метрика, где есть неочевидный термин, имеет иконку `Info` (lucide, 14px, серая, рядом с заголовком). При hover/tap открывается popover с тремя секциями:

```
┌──────────────────────────────────┐
│ Конверсия в заказ                │  ← название метрики
├──────────────────────────────────┤
│ Что это                          │
│ Какой процент посетителей сайта  │
│ оформил заказ.                   │
│                                  │
│ Как считается                    │
│ Кол-во заказов ÷ Уникальные      │
│ посетители × 100%                │
│                                  │
│ На что обращать внимание         │
│ Норма для магазина электроники   │
│ в РФ: 1–3%. Чем дороже товар,    │
│ тем ниже конверсия — это норм.   │
└──────────────────────────────────┘
```

- Тултипы реализуй на `@base-ui-react Popover` (не Tooltip — нужны клики на мобильных)
- Текст пояснений — из раздела 7 этого документа (глоссарий)
- На мобильных popover открывается тапом, закрывается тапом вне или по `X`
- Тон — дружественный, без жаргона, с практической пользой («норма», «на что влияет», «когда плохо»)

### 2.5. ChartCard — обёртка для всех графиков

```
┌────────────────────────────────────────────┐
│ Конверсионная воронка ⓘ      [⋯]            │  ← заголовок с тултипом + меню (экспорт/обновить)
│ Подзаголовок с контекстом                  │
├────────────────────────────────────────────┤
│                                            │
│        [Тело графика]                      │
│                                            │
└────────────────────────────────────────────┘
```

- Тонкая рамка (1px, `border-gray-200`), `rounded-2xl`, белый фон
- Тень `shadow-sm` на hover усиливается
- Меню `⋯` (lucide MoreHorizontal): «Экспорт PNG», «Экспорт CSV», «Обновить»

### 2.6. Empty states

**КРИТИЧНО:** дашборды на старте будут полупустыми (мало трафика, мало заказов). Не показывай мёртвые пустые блоки — показывай дружественные заглушки:

```
┌────────────────────────────────────────────┐
│       [иконка из lucide, 40px, серая]     │
│                                            │
│       Данных пока нет                      │
│                                            │
│       Метрика начнёт показываться, когда   │
│       посетители начнут пользоваться       │
│       поиском на сайте.                    │
│                                            │
│       [Подробнее о метрике →]              │
└────────────────────────────────────────────┘
```

- Иконка — релевантная контексту (`Search`, `ShoppingCart`, `Users`, `TrendingUp`)
- Текст всегда объясняет **почему пусто и когда появятся данные**
- Опциональная ссылка на пояснение метрики

### 2.7. Loading states

Не спиннеры — **skeleton-загрузка** для каждого виджета. Skeleton повторяет форму контента: KPI-плитка → 4 серых блока; график → серый прямоугольник с пульсацией; таблица → 5 строк-пустышек.

### 2.8. Цвета и стиль

- Серая палитра (как весь сайт), акцент — чёрный
- Положительные дельты: `#0a7d3e` (зелёный, приглушённый)
- Отрицательные дельты: `#c01818` (красный, приглушённый)
- Графики — только оттенки чёрного-серого, исключение: воронка (градиент от чёрного к серому), heatmap (от белого к чёрному через серый)
- НИКАКИХ радуг и многоцветных диаграмм — это магазин Apple
- Сетка графиков — `#e5e5e7`, очень тонкая

### 2.9. Адаптив

- На мобильном (< 768px) табы превращаются в горизонтально-скроллящуюся ленту
- KPI-плитки укладываются 2×N (с десктопа 6×1)
- Графики занимают 100% ширины, в одну колонку
- Таблицы → горизонтальный скролл с фиксированной первой колонкой

### 2.10. Анимации (сдержанно)

- Smooth transitions между табами (180ms, easeOut)
- Числа KPI плавно «накручиваются» при загрузке (использовать `motion` `animate` props, 600ms; respects `prefers-reduced-motion`)
- Sparkline рисуется stagger-ом (slight delay между точками)
- Никаких infinite-loop анимаций. Используй `fixing-motion-performance` для контроля

## 4. ФАЗА 2 — АНАЛИТИКА САЙТА (5 ТАБОВ)

URL: `/admin/analytics/site?tab=...`

### Таб 1 — Обзор (`?tab=overview`, по умолчанию)

Самое важное на одном экране. То что владелец смотрит первым делом утром.

**Сверху — ряд из 6 KPI:**
1. **Уникальные посетители** (с дельтой)
2. **Просмотры страниц** (с дельтой)
3. **Заявки** (формы trade-in + обратный звонок + вопросы)
4. **Конверсия в заказ** (%) — заказы / уник.посетители
5. **Средняя глубина просмотра** (страниц за сессию)
6. **Bounce rate** (% сессий с одним просмотром)

**Главный график:** просмотры по дням (LineChart) с двумя линиями — текущий и прошлый период. Hover показывает значение в точке.

**Снизу — две колонки:**
- Слева: **Конверсионная воронка** (FunnelChart, 6 шагов: Просмотр страницы → Просмотр товара → Добавление в корзину → Начало оформления → Отправка заказа → Оплата). На каждом шаге: число событий, % от 1-го, drop-off от предыдущего. Подсветить самый большой drop-off.
- Справа: **Источники трафика** (DonutChart + таблица): Прямые / Поиск (Я+G) / Соцсети / Реклама / Реферры / E-mail. Для каждого: визиты, конверсия в заказ, выручка.

### Таб 2 — Аудитория (`?tab=audience`)

Кто к нам приходит.

**Ряд KPI:**
1. **Новые посетители** — кто пришёл впервые
2. **Возвращающиеся** — кто был раньше
3. **% возврата** — возвращающиеся / все × 100%
4. **Средняя длительность сессии** (мин:сек)

**Виджеты:**
- **Новые vs Возвращающиеся** — DonutChart + динамика по дням (две тонкие линии)
- **Частота визитов** — гистограмма: 1 визит / 2-3 / 4-7 / 8+. Подсказка: «Здоровый магазин имеет 15-25% посетителей с 2+ визитами».
- **Гео по городам РФ** — `<GeoMap />` России с точками на городах + таблица топ-городов (визиты, заказы, конверсия по городу). Белгород выделен. Подсказка: «Понимание реального покрытия. Если много заказов из других городов — стоит расширить доставку».
- **Устройства** — DonutChart (desktop/mobile/tablet) + дельта vs прошлый период
- **Браузеры / ОС** — две мини-таблицы рядом, топ-5 каждая
- **Время до первой покупки (медиана)** — KpiCard + гистограмма распределения. Подсказка: «Медианное время от первого визита до оформления заказа. Длинный цикл = клиент долго думает; нужен ретаргетинг».

### Таб 3 — Поведение (`?tab=behavior`)

Что пользователи делают на сайте.

**Ряд KPI:**
1. **Поисковых запросов на сайте** — сколько раз использовали поиск
2. **Запросов без результатов** — критичная метрика для пополнения каталога
3. **Добавлено в избранное** (wishlist) — за период
4. **Брошенных корзин** — начатых и не завершённых

**Виджеты:**
- **Топ-страниц** (DataTable) — путь, просмотры, уник.посетители, среднее время на странице, bounce rate страницы, % переходов в корзину со страницы (только для карточек товара)
- **Страницы выхода** (DataTable) — где чаще всего уходят с сайта. Подсказка: «Страницы с самым высоким % выхода — слабые звенья. Карточки товара тут — норма, остальное — повод к проверке».
- **Поисковые запросы** — две колонки:
  - **С результатами** (топ-20)
  - **Без результатов** ⚠️ (топ-20, с бейджем-предупреждением). Подсказка: «Эти запросы — золото для закупа: люди ищут, а у вас этого нет в каталоге»

## 7. ГЛОССАРИЙ — ТЕКСТЫ ТУЛТИПОВ (готовые к копированию)

Каждый тултип имеет три секции: **Что это**, **Как считается**, **На что обращать внимание**. Используй их as is.

### Аналитика сайта

**Уникальные посетители**
- *Что это:* Количество разных людей, посетивших сайт за период. Один человек, зашедший несколько раз, считается одним.
- *Как считается:* По уникальному ID, который записывается в браузер посетителя.
- *На что обращать внимание:* Рост на 10-20% месяц к месяцу — хороший знак. Резкое падение — повод проверить рекламу и SEO.

**Просмотры страниц**
- *Что это:* Сколько раз страницы сайта были открыты за период.
- *Как считается:* Каждое открытие страницы — один просмотр. Если человек открыл главную, потом каталог, потом товар — это 3 просмотра.
- *На что обращать внимание:* Соотношение просмотров к посетителям = глубина просмотра. Норма — 3-5 страниц на сессию.

**Заявки**
- *Что это:* Сколько раз посетители заполнили формы на сайте (trade-in, обратный звонок, вопросы).
- *На что обращать внимание:* Заявки — это потенциальные клиенты. Обрабатывайте их в течение часа, пока интерес горячий.

**Конверсия в заказ**
- *Что это:* Какой процент посетителей сайта оформил заказ.
- *Как считается:* Кол-во заказов ÷ Уникальные посетители × 100%.
- *На что обращать внимание:* Норма для магазина электроники в РФ: 1-3%. Чем дороже товар, тем ниже конверсия — это нормально.

**Средняя глубина просмотра**
- *Что это:* Сколько страниц в среднем смотрит один посетитель за визит.
- *Как считается:* Просмотры ÷ Сессии.
- *На что обращать внимание:* Норма 3-5. Меньше 2 — сайт не вовлекает, люди уходят со входной страницы.

**Bounce rate (показатель отказов)**
- *Что это:* Доля посетителей, которые ушли после просмотра одной страницы.
- *На что обращать внимание:* Норма 40-55%. Больше 60% — страница не оправдывает ожиданий, или реклама приводит нецелевых посетителей. **Меньше = лучше.**

**Новые vs Возвращающиеся**
- *Что это:* Новые — кто впервые на сайте. Возвращающиеся — кто уже был раньше.
- *На что обращать внимание:* Здоровый магазин имеет 30-40% возвращающихся. Если все новые — нет удержания, нужны триггерные рассылки.

**Время до первой покупки**
- *Что это:* Сколько проходит от первого визита до первого заказа.
- *На что обращать внимание:* Электроника — долгий цикл, медиана 3-14 дней нормальна. Если очень длинный — нужен ретаргетинг.

**Запросы без результатов (поиск)**
- *Что это:* Что искали на сайте, но ничего не нашли.
- *На что обращать внимание:* Это золото для закупа. Если ищут «AirTag», а у вас его нет — добавьте в каталог. Если ищут «iPhone 12», но он у вас только в Б/У — переименуйте раздел или сделайте редирект.

**Hero CTR**
- *Что это:* Кликабельность баннера на главной. Из всех показов сколько было кликов.
- *Как считается:* Клики ÷ Показы × 100%.
- *На что обращать внимание:* Норма 1-3%. Меньше 1% — баннер не работает, нужно сменить картинку, заголовок или предложение.

**LCP / INP / CLS (Core Web Vitals)**
- *Что это:* Метрики скорости сайта от Google.
  - **LCP** — за сколько секунд загружается главное содержимое страницы (норма <2,5с)
  - **INP** — насколько быстро сайт реагирует на клики (норма <200мс)
  - **CLS** — насколько прыгают элементы при загрузке (норма <0,1)
- *На что обращать внимание:* Влияет на позиции в Google. Красные показатели — пользователи уходят, не дождавшись загрузки.

### Аналитика заказов

**Выручка**
- *Что это:* Общая сумма всех оплаченных заказов за период.
- *На что обращать внимание:* Сравнивайте с прошлым периодом и с тем же периодом год назад (учёт сезонности).

**Средний чек (AOV)**
- *Что это:* Сколько в среднем тратит один покупатель за заказ.
- *Как считается:* Выручка ÷ Кол-во заказов.
- *На что обращать внимание:* Рост AOV без потери конверсии — самый здоровый рост. Способы: кросс-сел (предложить чехол к iPhone), ап-сел (предложить 256GB вместо 128GB).

**LTV (Lifetime Value)**
- *Что это:* Сколько денег приносит клиент за всё время покупок в магазине.
- *Как считается:* Сумма всех заказов клиента.
- *На что обращать внимание:* Если LTV выше стоимости привлечения (CAC) — реклама окупается. Чем выше LTV — тем больше можно тратить на привлечение.

**Repeat purchase rate**
- *Что это:* Какой процент клиентов сделал второй+ заказ.
- *На что обращать внимание:* Норма для электроники: 15-25%. Низкий показатель = слабое удержание, нужны бонусы, триггеры, программа повторного выкупа.

**ABC-анализ**
- *Что это:* Распределение товаров по вкладу в выручку.
  - **A** — топ 20% товаров, делающих 80% выручки. Локомотивы.
  - **B** — средние 30%, дающие 15% выручки.
  - **C** — оставшиеся 50%, дающие 5% выручки. Балласт.
- *На что обращать внимание:* Группа A не должна заканчиваться. Группа C — кандидаты на вывод или акцию.

**XYZ-анализ**
- *Что это:* Стабильность спроса на товар.
  - **X** — стабильные продажи (например, AirPods каждый месяц одинаково). Безопасный запас.
  - **Y** — со скачками (iPhone — пик в сентябре).
  - **Z** — непредсказуемые. Рискованно закупать впрок.
- *На что обращать внимание:* Сочетание с ABC — самый сильный инструмент закупа.

**ABC×XYZ (матрица)**
- *Что это:* Сочетание ABC и XYZ. Главные группы:
  - **AX** — лидеры выручки со стабильным спросом. Локомотивы. Держать запас всегда.
  - **AZ** — много продаются, но непредсказуемо. Рисковая категория, держать малый запас.
  - **CZ** — мало продаются и непредсказуемо. Балласт, вывод или распродажа.
- *На что обращать внимание:* Каждая ячейка матрицы — отдельная закупочная стратегия.

**Оборачиваемость**
- *Что это:* На сколько дней хватит текущего остатка товара при текущей скорости продаж.
- *Как считается:* Остаток ÷ Средние дневные продажи.
- *На что обращать внимание:* Меньше 14 дней — пора заказывать. Больше 90 дней — товар «зависает», деньги заморожены.

**RFM-сегментация**
- *Что это:* Сегментация клиентов по 3 признакам:
  - **R** (Recency) — когда последний раз покупал (недавно = хорошо)
  - **F** (Frequency) — как часто покупает
  - **M** (Monetary) — на какую сумму
- *На что обращать внимание:* Не все клиенты одинаковые. Чемпионам — VIP-подход. Под угрозой — реактивация. Спящим — щедрая акция или забыть.

**Сегменты RFM:**
- **Чемпионы** — лучшие клиенты. Часто, недавно, много. Поощряйте бонусами, ранним доступом.
- **Лояльные** — стабильные постоянные клиенты. Удерживайте программой лояльности.
- **Под угрозой** — раньше покупали часто, но давно не возвращались. Срочно реактивация с персональной скидкой.
- **Спящие** — давно ничего не покупали. Последняя попытка вернуть через щедрую акцию.
- **Новые** — недавно сделали первую покупку. Welcome-серия писем, второй заказ — критически важен.

**Когортный анализ**
- *Что это:* Группировка клиентов по месяцу первой покупки и отслеживание их возвращаемости.
- *На что обращать внимание:* Если процент возвращающихся в новых когортах выше, чем в старых — качество привлекаемых клиентов растёт.

**Доля заказов с trade-in**
- *Что это:* Какой процент новых iPhone куплен через trade-in (с зачётом старого устройства).
- *На что обращать внимание:* Чем выше — тем сильнее программа trade-in работает на конверсию. Цель — 20-30%.

**ROI программы trade-in**
- *Что это:* Окупаемость программы trade-in с учётом перепродажи Б/У и upsell-эффекта.
- *Как считается:* (Выручка от продажи Б/У + дополнительная выручка от новых iPhone, купленных через trade-in − затраты на выкуп) ÷ затраты на выкуп × 100%.
- *На что обращать внимание:* Trade-in часто кажется убыточным сам по себе, но окупается через рост конверсии в новый iPhone. Считайте полный ROI.
