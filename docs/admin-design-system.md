# Дизайн-система админки PhoneTrade

Правила, чтобы формы и контролы выглядели аккуратно и единообразно (Apple-минимализм,
серо-чёрные токены). Нарушение этих правил = «уродский» UI, которого быть не должно.

## Токены и палитра
- Только токены: `bg-ink`, `text-ink`, `text-ink-muted`, `text-ink-subtle`, `border-border`,
  `bg-surface`, `bg-white`. Цветное исключение — `text-sale`/`bg-sale` (только смысловой акцент).
- Радиусы: контролы `rounded-sm`/`rounded-lg`, карточки/панели `rounded-xl`/`rounded-2xl`.
- Тени почти отсутствуют; для поповеров — мягкая диффузная тень.

## Иконки
- Выбор иконки — **только** через `IconPicker`: компактная квадратная кнопка, показывающая
  **саму иконку**. НИКОГДА не выводить kebab-имя иконки (`zap`, `map-pin`) как текст в UI.
- Очистка иконки — пунктом «Без иконки» внутри поповера, не крестиком на триггере.
- Единый набор — `src/lib/admin/icons.ts` (`ICON_SET`/`resolveIcon`), тематический (коммерция+техника).

## Формы и карточки настроек
- Поля — через `Field` + `TextInput`/`Textarea`/`Select`/`Switch` из `components/admin/form`.
- Карточка элемента списка: заголовок-строка `[иконка] [название (flex-1)] [switch] [↑][↓][🗑]`,
  ниже — поля в `grid` (sm:grid-cols-2 / `[1fr_200px]`). НИКАКИХ накладывающихся блоков и
  абсолютного позиционирования внутри строк формы.
- Тяжёлые загрузчики (`ImageField`) НЕ ставить в узкую колонку рядом с полями — выносить на
  отдельную строку или прятать в `<details>` («Своя иконка»), чтобы не ломать сетку.
- Компактность: не растягивать одну опцию на пол-экрана; группировать логично, дышать
  умеренно (p-4/p-5), не плодить лишние подписи-«хинты» под каждым контролом.
- Списки с добавлением: кнопка «Добавить …» (outline, sm) внизу; sticky-бар «Сохранить» снизу.

## Списки с элементами → строка + модалка (единый паттерн)
- Редактирование элементов списков (способы оплаты/доставки, опции, бейджи) — **единообразно**:
  компактная строка `[иконка] [название + подпись] [switch] [Изменить][↑][↓][🗑]`, по «Изменить»
  открывается **модалка** (`components/admin/Modal`) со всеми полями. НЕ городить все поля inline.
- Все модалки наследуют один вид: заголовок + контент + футер с кнопкой «Готово»; портал, z-[80].
- IconPicker в модалке — компактная иконка-кнопка; поповер показывает **все иконки сразу** (без
  внутреннего скролла), 8 колонок.

## Поповеры/выпадающие
- Только кастомные (в интерфейсе), не нативные `<select>` — `Select` из `components/admin/form`,
  меню в портале (не обрезается таблицами). z-индексы: поповеры `z-[60]/[61]`, модалки `z-[80]`.

## Чек-лист перед сдачей экрана админки
- [ ] Нет текстовых kebab-имён иконок в интерфейсе.
- [ ] Нет накладывающихся/обрезанных элементов; сетка ровная.
- [ ] Контролы и отступы — на токенах, компактно и единообразно.
- [ ] Выпадающие — кастомные; иконки — через IconPicker.
- [ ] Есть состояние пустого списка и понятные подписи.
